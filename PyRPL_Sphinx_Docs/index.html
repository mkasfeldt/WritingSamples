Welcome to PyRPL’s Documentation!¶

Intro to PyRPL:

The PyRPL module is a custom Python module that performs a variety of land records functions. The main functions are convenience functions that use internal functions in the __PyRPL folder. PyRPL uses python docstring to document usage it's functions. To access it use the name of the class or method and the __doc__ function.

PyRPL uses sqlite to house its data. This can be changed to use a more robust database, but sqlite meets our current needs. If we changed to another database, PostgreSQL should be used as it is thread safe by design. This will allow far fewer changes to the LRWS than if we use a non-thread safe database (i.e. MySQL)

Requirements:

Python 2.7 or newer of version 2 python must be installed

Copy the entire PyRPL folder to the Python2.xLibsite-pakages folder where x is the minor version of python you are using (minimum 2.7)

If using on Windows the path should be:
C:Python2.7Libsite-packages
If using on Ubuntu 10.04 LTS Linux server:
/usr/local/lib/python2.7/site-packages
